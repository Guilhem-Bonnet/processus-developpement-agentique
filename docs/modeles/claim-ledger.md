# Modèle de claim ledger

Le claim ledger relie les affirmations importantes à des preuves. Une affirmation sans preuve reste une hypothèse.

| ID | Affirmation | Type | Source ou preuve | Statut | Confiance | Décision |
| --- | --- | --- | --- | --- | --- | --- |
| CL-001 | A renseigner | fait / hypothèse / résultat / décision | chemin, commande, lien, capture | prouvé / hypothèse / contredit | faible / moyenne / élevée | utiliser / vérifier / rejeter |

## Types d'affirmations

| Type | Preuve minimale |
| --- | --- |
| Fichier ou code | chemin lu, diff ou extrait. |
| Test | commande et résultat. |
| API ou norme | contrat, documentation officielle ou code. |
| Design | charte, design system, maquette ou validation DA. |
| Sécurité | scan, règle, revue ou threat model. |
| Mémoire | source originale, date, score, portée. |
| Décision client | validation explicite ou ticket. |

## Synthèse

| Question | Réponse |
| --- | --- |
| Affirmations bloquantes non prouvées | A renseigner |
| Contradictions détectées | A renseigner |
| Hypothèses acceptées temporairement | A renseigner |
| Preuves à obtenir avant livraison | A renseigner |
